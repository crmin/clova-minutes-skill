# Learnings

## TL;DR

- README는 스킬 명세 자체보다 사람 기준의 이해와 다음 행동을 우선해야 한다.
- 이 저장소는 소개 문서, 설치 문서, 스킬 본문이 각각 다른 독자를 대상으로 한다.
- 테스트 샘플이 이미 있으므로 README에서 별도 장황한 설명보다 예시 파일 링크를 제공하는 편이 낫다.
- 설치는 사람이 `INSTALL.md`를 읽는 방식보다, README에서 설치 요청 문구를 복사해 에이전트에게 전달하는 방식이 더 자연스럽다.
- 에이전트에게 직접 전달할 코드블록은 README 본문 언어와 별개로 영어로 두는 편이 토큰 효율과 실행 일관성에 맞다.

## 근거

- `SKILL.md`는 실제 동작 규칙과 예외 처리 기준을 담고 있어 사람이 처음 읽기에는 밀도가 높다.
- `INSTALL.md`는 에이전트가 그대로 따라 할 수 있도록 설치 경로와 프롬프트 중심으로 작성되어 있다.
- `tests/` 아래에 템플릿, 전사본, 생성 결과가 모두 있어 README에서 결과 예시를 설명하기 좋다.

## 결론

- README는 "무엇을 해결하는가", "어떻게 설치를 시작하는가", "어떻게 요청하는가", "왜 신뢰할 수 있는가"에 집중해야 한다.
- `SKILL.md`와 `INSTALL.md`가 영어라는 내부 배경은 README에 노출하기보다 작업 문서 맥락에서만 다루는 편이 낫다.

## 재사용 키워드

- readme audience
- skill repository overview
- human-facing documentation
