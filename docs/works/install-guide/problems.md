# Problems

## 문제 정의

`install.draft.md`는 어떤 에이전트에 어떤 경로로 설치해야 하는지와 `references/`를 어떻게 처리해야 하는지를 충분히 설명하지 못했다. 특히 Codex와 Claude Code는 공개 문서, 예제, 생태계 관행이 서로 다른 경로를 보여 줄 수 있어 그대로 문서화하면 사용자가 혼동할 수 있었다.

## 원인

- 초안은 요구사항만 있고 실제 설치 경로 조사 결과가 없었다.
- Agent Skills 생태계와 각 제품의 네이티브 기능이 완전히 동일하지 않았다.

## 해결 내용

- `INSTALL.md`를 신규 작성해 공통 설치 원칙, 최종 디렉터리 구조, 에이전트별 권장 경로를 분리해 기록했다.
- 전역 설치를 기본 정책으로 다시 정리하고, 프로젝트 로컬 설치는 사용자 명시 요청이 있을 때만 허용하도록 문구를 수정했다.
- Codex와 Claude Code에는 경로 차이에 대한 보조 설명을 추가했다.
- 최종 배포 문서가 GitHub에서 직접 사용될 수 있도록 `INSTALL.md`를 영어로 전환했다.

## 회귀 방지

- 향후 에이전트별 설치 경로가 바뀌면 `INSTALL.md`뿐 아니라 `docs/works/install-guide/learnings.md`와 `decisions.md`도 함께 업데이트한다.
