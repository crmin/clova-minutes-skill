# Decisions

## Current (Active)

- `INSTALL.md`는 `SKILL.md`와 `references/`를 함께 복사하는 디렉터리 설치 문서로 작성한다. 상대 경로 참조가 있기 때문이다.
- 설치 기본값은 사용자 전역 스킬 디렉터리로 두고, 프로젝트 로컬 설치는 사용자가 명시적으로 요청할 때만 안내한다.
- 에이전트별 설치 경로는 공개 문서 또는 공개 카탈로그 기준으로 정리하고, 경로 차이가 있는 경우에는 호환 메모를 함께 둔다.
- `INSTALL.md`는 설치 절차만 담고, 배포 채널별 안내 문구는 별도 문맥에서 관리한다.
- 최종 `INSTALL.md` 본문은 GitHub 배포 문서이므로 영어로 유지한다.

## Change Log

### 2026-04-13

- Changed: `INSTALL.md`를 에이전트별 경로와 요청 문구 중심의 문서로 설계했다.
- Reason: 사용자는 수동 설치 문서가 아니라, 설치 요청을 받은 에이전트가 직접 따라 할 수 있는 가이드를 원했다.

### 2026-04-13

- Changed: Codex는 `.agents/skills`를 기본 경로로 두고 `~/.codex/skills`는 호환 메모로만 남기기로 했다.
- Reason: 공개 Codex 문서는 `.agents/skills`를 현재 탐색 경로로 설명하지만, 일부 로컬 도구와 예제는 `~/.codex/skills`를 여전히 사용한다.

### 2026-04-14

- Changed: 설치 안내 기본값을 전역 스킬 디렉터리로 통일하고, 프로젝트 로컬 설치는 명시적 요청이 있을 때만 허용하도록 문구를 수정했다.
- Reason: 사용자가 기대하는 기본 동작은 전역 설치이며, 프로젝트 로컬은 예외 경로로 다루는 편이 문서 역할에 더 맞는다.

### 2026-04-14

- Changed: `INSTALL.md` 본문을 영어로 전환했다.
- Reason: 설치 문서는 GitHub를 통해 다양한 에이전트와 환경에서 직접 참조될 가능성이 높아 영어가 더 적합하다.
