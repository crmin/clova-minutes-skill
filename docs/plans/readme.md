# readme 계획

## 목표

- 저장소의 목적과 사용 방식을 사람이 빠르게 이해할 수 있는 `README.md`를 작성한다.
- README 상단에 사람이 복사해서 에이전트에게 전달할 영문 설치 요청 문구를 배치한다.
- `README.md`는 한국어로 작성하고, 사람 기준의 소개와 설치 시작 방법에 집중한다.
- README는 배포 산출물 기준으로 작성하고, `.gitignore` 대상 파일이나 디렉터리에 의존하지 않는다.

## 범위

- `README.md` 신규 작성
- README 작성 배경과 결정 사항을 `docs/works/readme/`에 기록

## Out of scope

- `SKILL.md` 내용 변경
- `INSTALL.md` 설치 정책 변경
- 예제 파일 내용 수정

## 리스크와 가정

- README는 사람 중심 문서여야 하므로 사람이 직접 설치 문서를 읽으라고 안내하기보다 복붙용 설치 요청 문구를 먼저 제공해야 한다.
- 스킬 본문을 그대로 번역해 붙이면 소개 문서가 아니라 명세 문서처럼 보일 수 있다.
- `.gitignore`에 포함된 테스트 자산은 GitHub 배포본에 없을 수 있으므로 README 링크 대상으로 쓰면 안 된다.

## 완료 기준

- README만 읽어도 스킬의 목적, 입력, 결과물 특성과 설치 시작 방법을 이해할 수 있다.
- README는 한국어로 유지되고, 사람 기준의 소개 문서 역할에 집중한다.
- README 상단에 에이전트에게 전달할 영문 설치 요청 문구가 있다.
- README가 무시 파일이나 무시 디렉터리에 링크로 의존하지 않는다.
